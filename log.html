Bootstarlight Ver 2.1.1 released

18 Apr 2014

本周版本更新主要新增了针对Bootstrap的在线LESS编译功能。

具体更新如下：

新增在线LESS编译功能
加入了在线LESS编译的Bootstrap Magic，可以导入已定义好的LESS变量，下载并压缩Bootstrap的LESS变量或CSS文件。支持Bootstrap3.x和Bootstrap2.x。
新增JavaScript组件
针对日期选择器在表单中的情景，统一了DatetimePicker组件和DaterangePicker组件的样式以适配Bootstrap。
加入了文件上传组件Dropzone实例Demo。
新增DSS业务实例Demo
设计了标签页在Panel容器中的样式。
BUG修复
修复了表单组件中，纯文本情况下1像素的问题。
修复了左侧目录三级菜单下位置偏移的问题。
修复了左侧目录二级菜单鼠标移入导致文本换行的问题。
修复了Bootstrap 2.x各主题中二级悬浮导航宽度不正确的问题。
UI优化
优化了新框架中默认用户的图片。
优化了新框架中Panel配色方案。
调整了部分主题方案中的字体大小及导航间距。
其它细节优化。

Bootstarlight Ver 2.0.1 released

12 Apr 2014

此次版本更新主要新增了部分JavaScript组件Demo及Login页面并修复了上个版本中发现的BUG、同步升级了部分组件版本。

具体更新如下：

新增登录页面
DSS Demo新增了登录页面，全局HTML5+CSS3动画，采用Long Shadows Generator长投影及响应式设计。用户头像可根据账号进行切换（usename中输入franki，当进一步输入密码时即可看到效果）。
新增JavaScript组件Demo
针对Choosen在表单中的情景，设计了一套兼容Bootstrap 3.1.1的Chosen UI。包括单项选择，多项选择，带搜索的选择，左右布局等。
新增DSS业务实例Demo
针对DSS中的Location Setup - Assign Affiliate单独设计了一套样式以兼容新系统。
BUG修复
修复了Animate.css在单独组件中的动画效果。
修复了新框架中警告框在小分辨率情况下导致警告文本换行，从而造成左侧图标错位及背景高度不全的问题。
修复了部分页面的链接及重定向问题。
修复了Panel组件panel-heading中添加按钮或按钮组导致高度被撑高的问题。
修复了loading页面的动画问题。
UI优化
优化了新框架中的表单布局结构。
优化了新框架中的面包屑样式。
优化了部分png文件的体积大小。
组件更新
更新了Animate.css。
更新了glyphicons字体。

Bootstarlight Ver 1.0 released

9 Apr 2014

这些天将基本的常用的Bootstrap的组件抽离出来，整到了一个页面。

具体更新如下：

30套各具特色的UI样式
其中14套基于Bootstrap2.3.2和16套基于Bootstrap3.1.1样式的主题，涵盖目前主流设计风格。包括默认的风格、暗夜蓝、光之影、海天蓝、黑色系、极简红、经典银、喇嘛红、老年版、商务蓝、时尚粉、乌邦图、西部黄、卡巴绿、质感黑、Metro风和Flat UI等。
直观的代码介绍
对每个单独的CSS组件都可以迅速查看源代码。
焕然一新的DSS框架
针对DSS业务开发中遇到的一些问题，简化了对应的语义结构和样式冗余，抛弃了多余且不常用的功能，加入了 animate.css 更具科技感。
编码规范
依据 W3C标准、 Code Guide 、Idiomatic CSS 和 Bootstrap官方文档 整理制作了Bootstrap Ver 3.1.1编码规范。
新增新框架表单实例
正对DSS业务开发制作的表单demo，包括内容布局，子项与子子项的排版，按钮位置，表单验证等。

New StarLight UI Kit plan

24 Mar 2014

在上次版本的StarLight UI Kit Ver 3.8版本的使用情况来看，实际用途不大，在未能熟悉原生Bootstrap的情况下，使用此框架务必会造成较大问题，主要是结构性的问题，因为此框架自定义了许多自己的结构和语法。考虑到从实用性出发，此次更新对StarLight UI Kit进行了重构，尽可能的使用原生Boostrap的语义结构和栅格系统，再在此基础上丰富UI展现。

具体重构计划如下：

全新LESS编译引擎
使用less编译样式，可以提高样式的维护性。更加符合Bootstrap编码规范。
抽离多余样式
直接引用StarLight UI Kit样式会造成代码冗余，性能低下。实际项目中并不会全部使用其中的样式。
更好的利用原生Bootstrap语义结构
简单、清晰、高效的语义结构有助于开发了解并熟悉Bootstrap的语义结构。
DSS业务适配
针对SMS-Assist DSS业务模型中的具体问题进行UI定义，更加的兼容DSS。
组件整合
调整JavaScript插件的整合方式，不再在框架中进行体现。单独设置页面进行介绍。
丰富UI展现
设计更多的Bootstrap基本样式框架，无论是使用2.3.2版本还是最新的3.1.1版本。
文档说明
对每个单独的CSS组件都可以迅速查看源代码，快速学习Bootstrap。

对于新框架计划，命名为Bootstarlight。
